1) Introdução

E ai galerinha DEV, de boas? Então, estive pensando aqui, uma ideia que eu sempre tive, e vivo brisando quando ando na rua:

		"Imagina se andar na rua fosse uma função?? Seria bem complexa..."

Pois bem, pensando nisso, e também pensando em praticar o desenvolvimento comunitário, eu resolvi criar esse projetinho aqui, chamado "walking.py", para tentar chegar o mais perto disso kkk

As motivações principais são:

	- Fazer um projeto "comunitário-solo"
	- Aprender a utilizar o GitHub
	- Desenvolver habilidades em Python
	- Aprender a aprender
	- Simular problemas de um projeto real

2) Execução

Eu estou disponibilizando o arquivo na versão 0_0_0 (que estará comentado no arquivo), a próxima vez que alguém alterar, ele deve ir para a versão 0_0_1, e assim suscetivamente. Quando acabar o primeiro ciclo (quando todos editarem uma vez) ela passa para a versão 0_1_0, e assim vai. Lembre-se de alterar no "header" do arquivo. Quando o arquivo chegar à versão 1_0_0 penso que seria um bom momento para nos comunicarmos, tão somente quando chegar à essa versão! Para que a 1_0_1 já seja mais alinhada.

Toda vez que você abrir o arquivo "walking.py", você deve incrementa-lo de alguma forma à tornar a função mais parecida com "andar na rua de verdade". Não precisam ser muitas linhas, nem necessariamente algo mega complexo, mas adicione "funcionalidades" interesasntes ao código. Inclusive, não faça muitas linhas mesmo, para que o próximo dev possa entender o que está acontecendo sem que você precise falar e ele possa continuar a incrementação de boa.

Comentários no código são permitidos, mas procure não dar "tudo de bandeja".

O ideal seria utilizar o GitHub pela linha de comando (como um dev pro), mas a princípio estou utilizando a interface do site. Quando conseguir, vou particularmente utilizar a CLI.

Embora vá contra os princípios básicos de um projeto em comunidade, pensei em #não nos comunicarmos# e por que?

Porque a idéia seria olhar o código e entende-lo sem ajuda, mesmo que com alguma classe, sintaxe ou etc que você ainda não conhece. Se não conhece, pesquisa > daí você conhece e pode incrementar aquela função.

Mudanças drásticas no código podem acontecer, desde que façam sentido, e não estrague tudo.

Haverá uma ordem na execução do projeto, que estará no header. A primeira rodada é livre, então quem quiser pode já começar, e esperar que outro assuma a vez. Uma vez estipulada a ordem dos devs, ela estará no header. Não é permitido editar o arquivo se não for a sua vez!! Aguarde, ou cobre o amiguinho carinhosamente  :)

Todo comentário e sugestão é válida, tudo pelo aprendizado!
